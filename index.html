Undoing Changes - this is used when we want to undo changes.

case 1: staged changes
git reset fileName - used this command when reset one in file
git reset - used this command when reset all last added changes
- stated changes means only add but not commit

case 2: commited changes (for one commit)
git reset HEAD~1 -> hear HEAD~1 means ((last commit is HEAD~1)) undo last commit - used only for single commit

case 3: commited changes (for multiple commits)
git reset <commit hash>
- here hash means commit id or commit hash. if we want back from commit 3 to commit 1 then we need to use commit hash of commit 1 hash
that hash we can check or get using git log command.

git reset --hard <commit hash> - using this command we can undo multiple commits at once. and that remove all the commit history from vs code and git also --hard is used with reset.

git log - used this command to see all commit history with commit id/hash
